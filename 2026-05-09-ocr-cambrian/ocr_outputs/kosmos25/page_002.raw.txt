<bbox><x_85><y_105><x_431><y_123></bbox>such as Visual FUDGE [ 6 ]. In this work, we tackle form
<bbox><x_85><y_123><x_431><y_140></bbox>preparation rather than understanding, and cast the task as
<bbox><x_85><y_140><x_431><y_158></bbox>one of object detection. That is, we do not attempt to extract
<bbox><x_85><y_157><x_431><y_175></bbox>values, but to predict where on the page the slots for values
<bbox><x_85><y_175><x_315><y_192></bbox>should be, based on visual information.
<bbox><x_85><y_221><x_341><y_240></bbox>Object Detection in Document Images
<bbox><x_355><y_221><x_431><y_239></bbox>Many docu-
<bbox><x_85><y_239><x_431><y_256></bbox>ments are inherently multimodal, so there are many estab-
<bbox><x_85><y_256><x_431><y_274></bbox>lished lines of research that cast document problems as vi-
<bbox><x_85><y_274><x_431><y_291></bbox>sion problems, and speciﬁcally object detection. Examples
<bbox><x_85><y_291><x_431><y_308></bbox>include layout detection (DocLayNet [ 19 ], PubLayNet [ 27 ],
<bbox><x_85><y_308><x_431><y_326></bbox>Newspaper Navigator [ 14 ], PRImA Layout [ 3 ] ), table de-
<bbox><x_85><y_326><x_431><y_343></bbox>tection (TableBank [ 16 ]), and math formula detection (For-
<bbox><x_85><y_343><x_431><y_361></bbox>mulaNet [ 20 ]). LayoutParser [ 21 ] is a suite of tools that
<bbox><x_85><y_360><x_431><y_378></bbox>provides a uniﬁed interface for all of these tasks. Depend-
<bbox><x_85><y_378><x_431><y_395></bbox>ing on the resolution required, models such as YOLO [ 11 ],
<bbox><x_85><y_395><x_431><y_413></bbox>the Document Image Transformer (DIT) [ 15 ], or Lay-
<bbox><x_85><y_413><x_430><y_430></bbox>outLM [ 25 ], for multimodality, have been used. C OMMON -
<bbox><x_85><y_430><x_431><y_447></bbox>F ORMS adopts the same paradigm for detecting form ﬁelds.
<bbox><x_85><y_447><x_431><y_465></bbox>In this work, we treat documents as unimodal images and
<bbox><x_85><y_465><x_431><y_482></bbox>train object detectors to localize and type form ﬁelds. The
<bbox><x_85><y_482><x_431><y_500></bbox>outputs can be used as a complementary semantic layer in
<bbox><x_85><y_500><x_265><y_517></bbox>addition to these other models.
<bbox><x_85><y_545><x_348><y_564></bbox>Document Corpora from Common Crawl
<bbox><x_362><y_546><x_431><y_563></bbox>ccPDF [ 22 ]
<bbox><x_85><y_563><x_431><y_580></bbox>and FinePDFs [ 13 ] curate PDF corpora from Common
<bbox><x_85><y_580><x_431><y_598></bbox>Crawl. Both datasets target visually and/or topically diverse
<bbox><x_85><y_598><x_431><y_615></bbox>datasets. Like these efforts, C OMMON F ORMS is also drawn
<bbox><x_85><y_615><x_431><y_633></bbox>from Common Crawl. However, in this work, the ﬁltering
<bbox><x_85><y_633><x_431><y_650></bbox>and preparation is focused on mining well-annotated forms
<bbox><x_85><y_650><x_431><y_667></bbox>which can be used as training signal for form ﬁeld detection.
<bbox><x_85><y_703><x_297><y_726></bbox>3. C OMMON F ORMS Dataset
<bbox><x_85><y_735><x_431><y_753></bbox>There is truth to the adage “quantity has a quality all its
<bbox><x_85><y_753><x_431><y_770></bbox>own”. The core thesis of this paper is twofold: (1) there are
<bbox><x_85><y_770><x_431><y_788></bbox>plentiful existing prepared forms in Common Crawl, and (2)
<bbox><x_85><y_788><x_431><y_805></bbox>those forms are high-quality enough to be used as a train-

<bbox><x_85><y_840><x_431><y_857></bbox>Our goal is to ﬁlter to forms that have been well-prepared.
